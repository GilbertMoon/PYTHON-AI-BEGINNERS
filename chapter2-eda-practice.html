EDA 종합 실습

데이터 불러오기, 전처리, 기초 통계, 시각화를 하나의 분석 흐름으로 연결합니다.

Chapter 2이전세미프로젝트

1. EDA란?

EDA는 Exploratory Data Analysis의 약자로 탐색적 데이터 분석을 의미합니다. 본격적인 모델링이나 의사결정 전에 데이터의 구조, 품질, 패턴, 이상값을 파악하는 단계입니다.

구조 확인행/열, 컬럼명, 자료형 확인
품질 확인결측치와 이상값 점검
통계 확인평균, 합계, 분포 확인
시각화패턴과 비교 결과 표현

2. EDA 기본 흐름

  1. 데이터를 불러온다.
  2. 데이터 구조를 확인한다.
  3. 결측치와 자료형을 확인한다.
  4. 기초 통계값을 확인한다.
  5. 중요 컬럼을 기준으로 집계한다.
  6. 그래프로 패턴을 확인한다.
  7. 분석 결과를 문장으로 정리한다.

3. 기본 코드 흐름

import pandas as pd import matplotlib.pyplot as plt df = pd.read_csv("sales.csv") print(df.head()) print(df.info()) print(df.isna().sum()) print(df.describe())

4. 집계와 시각화 연결

category_summary = df.groupby("category")["sales"].sum() print(category_summary) category_summary.plot(kind="bar") plt.title("카테고리별 총매출") plt.xlabel("카테고리") plt.ylabel("총매출") plt.show()

집계 결과를 바로 그래프로 연결하면 분석 내용을 더 쉽게 설명할 수 있습니다.

5. 분석 문장 작성

EDA 결과는 코드와 그래프만으로 끝나면 안 됩니다. 관찰한 내용을 문장으로 정리해야 합니다.

관찰해석 문장
커피 카테고리 매출이 가장 높다카페 매출의 핵심은 커피 메뉴로 보이며, 커피 메뉴 관리가 중요합니다.
특정 날짜 매출이 낮다해당 날짜의 운영 이슈나 날씨, 휴무 여부를 확인할 필요가 있습니다.
결측치가 존재한다분석 전 데이터 수집 과정 또는 입력 누락 여부를 점검해야 합니다.

6. 실습 체크리스트

  • EDA의 목적을 설명할 수 있다.
  • 데이터 구조와 결측치를 확인할 수 있다.
  • 기초 통계값을 확인할 수 있다.
  • 그룹별 집계를 수행할 수 있다.
  • 집계 결과를 그래프로 표현할 수 있다.
  • 분석 결과를 문장으로 정리할 수 있다.

7. 종합 실습 문제

1. 문제 출제

카페 매출 데이터에 대해 EDA 리포트를 작성하세요.

  • 데이터 구조를 확인합니다.
  • 결측치 개수를 확인합니다.
  • 전체 매출 합계와 평균을 계산합니다.
  • 카테고리별 총매출을 계산합니다.
  • 카테고리별 총매출 막대그래프를 그립니다.
  • 분석 결과를 3문장으로 정리합니다.

2. 개념 요약

개념활용
head/info구조 확인
isna결측치 확인
groupby카테고리별 집계
plot그래프 작성

3. 수도코드

CSV 파일을 읽는다. 앞부분 데이터와 자료형을 확인한다. 결측치 개수를 확인한다. 전체 매출 합계와 평균을 계산한다. 카테고리별 총매출을 계산한다. 막대그래프로 시각화한다. 분석 결과 문장 3개를 작성한다.

4. 실행 예시

전체 매출: 235000 평균 매출: 39166.67 카테고리별 총매출: coffee 127000 ade 30000 food 38000 분석 문장: 1. coffee 카테고리의 매출 비중이 가장 큽니다. 2. food 카테고리는 거래 수는 적지만 단가가 높을 수 있습니다. 3. ade 카테고리는 추가 프로모션 대상으로 검토할 수 있습니다.
