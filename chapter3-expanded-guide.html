1. Chapter 3의 핵심 관점

Chapter 3에서는 AI와 머신러닝을 어려운 수학 모델로 접근하기보다, 데이터를 이용해 예측 규칙을 만드는 과정으로 이해하는 것이 중요합니다. 입문자에게 가장 중요한 질문은 “어떤 모델을 쓸까?”가 아니라 “무엇을 예측하고 싶은가?”입니다.

같은 고객 데이터라도 다음 달 구매금액을 예측하면 회귀 문제가 되고, 구매 여부를 예측하면 분류 문제가 됩니다. 따라서 머신러닝 프로젝트는 항상 문제 정의에서 시작해야 합니다.

문제 정의무엇을 예측할지 정합니다.
입력값 X예측에 사용할 단서를 정합니다.
목표값 y모델이 맞혀야 할 값을 정합니다.
평가와 해석모델 결과를 업무 문장으로 설명합니다.

2. 지도학습을 더 쉽게 이해하기

지도학습은 입력값과 정답이 함께 있는 데이터로 학습하는 방식입니다. 모델은 입력값과 정답 사이의 관계를 찾아 새로운 입력값이 들어왔을 때 결과를 예측합니다.

업무 상황입력값 X목표값 y문제 유형
광고비로 매출 예측광고비, 방문자 수매출회귀
고객 구매 여부 예측방문횟수, 장바구니 금액구매/미구매분류
수강생 수료 여부 예측출석률, 과제점수수료/미수료분류
강의 포인트: 결과가 숫자 금액이나 점수처럼 연속값이면 회귀, 결과가 정해진 라벨이면 분류라고 설명하면 됩니다.

3. 회귀 보강

회귀는 숫자를 예측하는 문제입니다. 회귀 모델은 입력값이 커지거나 작아질 때 목표값이 어떻게 변하는지 학습합니다. 가장 단순한 예시는 광고비가 늘수록 매출이 증가하는 관계입니다.

  1. 목표값이 숫자인지 확인합니다.
  2. 입력값과 목표값의 관계가 있을 것 같은지 생각합니다.
  3. 예측값과 실제값을 비교합니다.
  4. 오차가 큰 데이터가 어떤 데이터인지 확인합니다.
  5. MSE 같은 평가 지표를 업무 단위로 해석합니다.
X = df[["ad_cost", "visitors"]] y = df["sales"] model.fit(X_train, y_train) pred = model.predict(X_test)

MSE는 오차를 제곱해 평균낸 값이므로 큰 오차에 민감합니다. “작을수록 좋다”는 것과 “단독으로 보기보다 다른 모델이나 기준과 비교해야 한다”는 점을 함께 설명합니다.

4. 분류 보강

분류는 정해진 클래스 중 하나를 예측하는 문제입니다. 구매/미구매, 합격/불합격, 정상/불량처럼 결과가 범주라면 분류 문제로 볼 수 있습니다.

정확도는 전체 중 맞힌 비율입니다. 하지만 어떤 종류의 오류가 발생했는지는 정확도만으로 알기 어렵습니다. 그래서 혼동행렬을 함께 확인합니다.

from sklearn.metrics import accuracy_score, confusion_matrix accuracy = accuracy_score(y_test, pred) cm = confusion_matrix(y_test, pred) print(accuracy) print(cm)
확인 항목해석 질문
정확도전체 중 얼마나 맞혔는가?
혼동행렬어떤 유형을 자주 틀렸는가?
예측 확률모델이 얼마나 확신하고 있는가?

5. 훈련 데이터와 테스트 데이터

모델이 학습 데이터에서만 잘 맞는 것은 충분하지 않습니다. 실제로 중요한 것은 처음 보는 데이터에서도 잘 작동하는지입니다. 그래서 데이터를 훈련 데이터와 테스트 데이터로 나눕니다.

훈련 성능테스트 성능해석
매우 높음낮음오버피팅 의심
낮음낮음학습 부족 또는 변수 부족
적절히 높음비슷하게 높음일반화 가능성 있음

6. 결과 해석 문장 만들기

모델은 테스트 데이터에서 [평가 지표]의 성능을 보였습니다. 이 결과는 [업무적 의미]로 해석할 수 있습니다. 다만 [한계점]이 있으므로, 실제 적용 전 [개선 방향]이 필요합니다.

예를 들어 “모델은 테스트 데이터에서 정확도 0.8의 성능을 보였습니다. 이는 고객 구매 여부를 어느 정도 구분할 수 있다는 의미입니다. 다만 데이터 수가 적으므로 실제 적용 전 더 많은 고객 데이터로 평가를 반복해야 합니다.”처럼 작성합니다.

8. Chapter 3 확장 실습 전략

  1. 업무 사례를 보고 회귀/분류를 먼저 구분합니다.
  2. 입력값 X와 목표값 y를 표로 정리합니다.
  3. 기본 모델을 학습합니다.
  4. 평가 지표를 출력합니다.
  5. 좋은 결과와 나쁜 결과 모두 해석 문장을 작성합니다.
  6. 결과의 한계와 개선 방향을 작성합니다.
Chapter 3 핵심 정리: 머신러닝은 모델 이름을 많이 아는 것이 목표가 아닙니다. 문제를 정의하고, 입력값과 목표값을 구분하고, 학습과 평가를 거쳐 결과를 업무적으로 해석하는 흐름을 이해하는 것이 핵심입니다.
