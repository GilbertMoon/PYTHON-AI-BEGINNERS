Chapter 4-5

미니 프로젝트 1: 문제 정의와 데이터 이해

최종 미니 프로젝트를 시작하기 전에 문제를 정의하고, 데이터셋과 컬럼의 의미를 파악합니다.

Chapter 4이전다음

1. 프로젝트는 문제 정의에서 시작합니다

분석 프로젝트에서 가장 중요한 첫 단계는 “무엇을 알고 싶은가?”를 명확히 하는 것입니다. 데이터부터 열어보는 것이 아니라, 데이터로 해결할 질문을 먼저 정리해야 합니다.

문제 정의분석으로 답하고 싶은 질문
데이터 선정문제 해결에 필요한 데이터
목표 설정최종 산출물의 기준
컬럼 이해각 데이터가 의미하는 바

2. 좋은 분석 질문 만들기

나쁜 질문좋은 질문
데이터를 분석해 보자어떤 메뉴가 매출에 가장 많이 기여하는가?
고객을 알아보자방문횟수와 구매금액 기준으로 고객을 나눌 수 있는가?
AI 모델을 만들어 보자고객 행동 데이터로 구매 여부를 예측할 수 있는가?
핵심: 좋은 질문은 분석 방법을 자연스럽게 결정하게 만듭니다.

3. 데이터셋 선택 기준

  • 행과 열 구조가 명확한 데이터인지 확인합니다.
  • 분석 목표와 관련된 컬럼이 있는지 확인합니다.
  • 결측치가 너무 많지 않은지 확인합니다.
  • 초보자 실습에서는 5~10개 컬럼 정도가 적절합니다.
  • 회귀 또는 분류 모델을 만들려면 목표값이 있어야 합니다.

4. 컬럼 정의서 만들기

컬럼 이름만 보고 바로 분석하기 어렵기 때문에, 각 컬럼의 의미를 정리하는 표를 먼저 만듭니다.

컬럼명의미자료형분석 활용
visit_count방문 횟수숫자고객 활동성
cart_amount장바구니 금액숫자구매 가능성
purchased구매 여부0/1분류 목표값

5. 실습 체크리스트

  • 분석 질문을 한 문장으로 작성할 수 있다.
  • 데이터셋 선택 이유를 설명할 수 있다.
  • 컬럼 정의서를 작성할 수 있다.
  • 회귀/분류/군집화 중 어떤 문제인지 판단할 수 있다.

6. 종합 실습 문제

1. 문제 출제

카페 고객 데이터를 사용해 미니 프로젝트 기획서를 작성하세요.

  • 분석 질문 1개를 작성합니다.
  • 사용할 데이터 컬럼 5개를 정리합니다.
  • 각 컬럼의 의미와 자료형을 작성합니다.
  • 분석 결과로 무엇을 만들지 작성합니다.

2. 문제 해결에 필요한 개념 요약

개념활용 방법
문제 정의분석 질문을 명확히 합니다.
컬럼 정의데이터의 의미를 확인합니다.
산출물 정의분석 결과의 형태를 정합니다.

3. 수도코드

분석하고 싶은 업무 상황을 정한다. 답하고 싶은 질문을 한 문장으로 작성한다. 필요한 컬럼을 선택한다. 각 컬럼의 의미와 자료형을 정리한다. 최종 산출물을 정의한다. 분석 성공 기준을 작성한다.

4. 실행 예시

분석 질문: 고객 행동 데이터로 구매 가능성이 높은 고객을 찾을 수 있는가? 사용 컬럼: visit_count, cart_amount, previous_orders, membership, purchased 최종 산출물: 구매 예측 모델 결과와 고객 그룹별 마케팅 제안
