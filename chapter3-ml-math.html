Chapter 3-2

머신러닝 수학 직관

벡터, 행렬, 함수, 시그모이드, 소프트맥스를 수식 암기가 아니라 데이터 흐름 관점에서 이해합니다.

Chapter 3이전다음

1. 수학을 왜 배우는가?

머신러닝의 수학은 어려운 공식을 외우기 위한 것이 아니라, 데이터가 모델 안에서 어떻게 계산되는지 이해하기 위한 언어입니다.

벡터여러 숫자를 한 줄로 묶은 데이터
행렬여러 행과 열을 가진 데이터
함수입력을 받아 출력을 만드는 규칙
확률값분류 모델의 예측 결과 표현

2. 벡터와 행렬

한 사람의 데이터가 나이, 구매횟수, 방문횟수라면 이 세 값은 하나의 벡터로 볼 수 있습니다. 여러 사람의 벡터가 모이면 행렬이 됩니다.

import numpy as np x = np.array([25, 3, 10]) X = np.array([[25, 3, 10], [32, 5, 8], [40, 2, 4]]) print(x) print(X)

3. 함수와 모델

모델은 입력값을 받아 예측값을 출력하는 함수처럼 생각할 수 있습니다.

def predict_sales(visit_count): return 10000 + visit_count * 3000 print(predict_sales(5))

실제 머신러닝은 이 함수의 규칙을 사람이 정하는 것이 아니라 데이터로부터 찾습니다.

4. 시그모이드와 소프트맥스 직관

개념역할쉬운 설명
시그모이드0~1 사이 값 출력이탈 확률, 구매 확률처럼 표현
소프트맥스여러 클래스 확률 출력A/B/C 중 어디에 가까운지 표현
핵심: 분류 모델은 보통 “정답 라벨”을 바로 찍기보다 각 클래스일 가능성을 계산한 뒤 가장 높은 값을 선택합니다.

5. 실습 체크리스트

  • 벡터와 행렬의 차이를 설명할 수 있다.
  • 모델을 함수처럼 이해할 수 있다.
  • 시그모이드가 확률처럼 해석되는 이유를 설명할 수 있다.
  • 소프트맥스가 다중 분류에서 쓰이는 이유를 설명할 수 있다.

6. 종합 실습 문제

1. 문제 출제

고객 3명의 나이, 방문횟수, 구매횟수를 행렬로 만들고, 간단한 규칙 함수로 예상 구매금액을 계산해 보세요.

2. 개념 요약

개념활용
행렬고객 여러 명의 입력 데이터 저장
함수구매금액 계산 규칙 표현
반복문고객별 예측값 출력

3. 수도코드

고객 데이터를 2차원 배열로 만든다. 방문횟수와 구매횟수를 이용하는 예측 함수를 만든다. 고객 데이터를 반복한다. 각 고객의 예측 구매금액을 계산한다. 결과를 출력한다.

4. 실행 예시

고객 1 예상 구매금액: 24000 고객 2 예상 구매금액: 31000 고객 3 예상 구매금액: 18000
