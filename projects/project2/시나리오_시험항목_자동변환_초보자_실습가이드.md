# 시험항목 기준 ↔ 시나리오 기준 자동 변환 실습 가이드

## 1. 실습 개요

이 문서는 방산·시험평가 업무에서 자주 발생하는 엑셀 정리 문제를 예제로 만든 초보자용 실습 가이드입니다.

현업에서는 시험항목이 수십 개에서 수백 개까지 존재하고, 하나의 시나리오에 여러 시험항목이 연결됩니다. 반대로 하나의 시험항목이 여러 시나리오에서 반복 사용될 수도 있습니다.

따라서 같은 데이터를 다음 두 가지 관점으로 관리해야 하는 경우가 많습니다.

| 구분 | 설명 |
|---|---|
| 시험항목 기준 | 시험항목 순번을 기준으로 어떤 시나리오에 포함되는지 보는 방식 |
| 시나리오 기준 | 시나리오 식별자를 기준으로 어떤 시험항목들이 포함되는지 보는 방식 |

이번 실습의 목표는 다음 두 가지입니다.

1. **시연 1**: 시험항목 기준 입력 파일을 읽어서 시나리오 기준 출력 파일로 자동 변환한다.
2. **시연 2**: 시나리오 기준 입력 파일을 읽어서 시험항목 기준 출력 파일로 자동 변환한다.

## 4. 핵심 개념 먼저 이해하기

### 4.1 엑셀의 보기용 공란 문제

현업 엑셀에서는 같은 대분류나 중분류를 반복해서 쓰지 않고, 아래처럼 첫 행에만 쓰는 경우가 많습니다.

| 순번 | 대분류 | 중분류 | 소분류 |
|---:|---|---|---|
| 1 | 운용환경 | 기동조건 | 저속 주행 |
| 2 |  |  | 중속 주행 |
| 3 |  | 통신조건 | 통신 연결 |

사람은 빈칸을 보고도 위 값과 같은 의미라고 이해할 수 있습니다. 하지만 파이썬은 빈칸을 실제 빈값으로 처리합니다.

그래서 자동 변환 전에 아래처럼 빈칸을 채워야 합니다.

| 순번 | 대분류 | 중분류 | 소분류 |
|---:|---|---|---|
| 1 | 운용환경 | 기동조건 | 저속 주행 |
| 2 | 운용환경 | 기동조건 | 중속 주행 |
| 3 | 운용환경 | 통신조건 | 통신 연결 |

이때 사용하는 pandas 기능이 `ffill()`입니다.

```python
normalized_df["대분류"] = normalized_df["대분류"].ffill()
normalized_df["중분류"] = normalized_df["중분류"].ffill()
```

`ffill()`은 forward fill의 줄임말입니다. 바로 위에 있는 값을 아래 빈칸으로 채운다는 뜻입니다.

---

### 4.2 정규화 데이터와 보기용 데이터

자동화에서 중요한 점은 **계산용 데이터**와 **보기용 데이터**를 구분하는 것입니다.

| 구분 | 설명 |
|---|---|
| 정규화 데이터 | 모든 행에 대분류, 중분류, 시나리오 식별자가 채워져 있는 데이터 |
| 보기용 데이터 | 사람이 보기 좋도록 반복되는 값을 다시 빈칸으로 처리한 데이터 |

파이썬 내부에서는 정규화 데이터로 계산하고, 엑셀에 저장할 때만 보기용 데이터로 바꾸는 방식이 안전합니다.

---

### 4.3 두 시연의 차이

| 구분 | 시연 1 | 시연 2 |
|---|---|---|
| 변환 방향 | 시험항목 기준 → 시나리오 기준 | 시나리오 기준 → 시험항목 기준 |
| 입력 기준 | 시험항목 순번 | 시나리오 식별자 |
| 주로 채우는 공란 | 대분류, 중분류 | 시나리오 식별자, 시나리오명, 운용조건 |
| 추가로 필요한 데이터 | 시나리오 코드표 | 시험항목 마스터 |
| 핵심 정렬 기준 | 시나리오 식별자 → 순번 | 순번 → 시나리오 식별자 |
| 핵심 함수 | `ffill()`, `merge()`, `sort_values()` | `ffill()`, `merge()`, `sort_values()` |

# PART A. 시연 1 — 시험항목 기준 → 시나리오 기준

## A-1. 시연 1의 업무 상황

입력 파일은 시험항목 기준으로 작성되어 있습니다.

예를 들어 아래와 같은 형태입니다.

| 순번 | 대분류 | 중분류 | 소분류 | 시나리오 식별자 |
|---:|---|---|---|---|
| 1 | 운용환경 | 기동조건 | 저속 주행 | SCN-MOB-001 |
| 2 |  |  | 중속 주행 | SCN-MOB-001 |
| 3 |  | 통신조건 | 통신 연결 | SCN-COM-001 |

이 데이터를 시나리오 기준으로 바꾸면 아래처럼 됩니다.

| 출력순번 | 시나리오 식별자 | 시나리오명 | 시험항목 순번 | 중분류 | 소분류 |
|---:|---|---|---:|---|---|
| 1 | SCN-COM-001 | 통신망 구성 시나리오 | 3 | 통신조건 | 통신 연결 |
| 2 | SCN-MOB-001 | 기동 운용 시나리오 | 1 | 기동조건 | 저속 주행 |
| 3 |  |  | 2 |  | 중속 주행 |

즉, 기준이 **시험항목 순번**에서 **시나리오 식별자**로 바뀝니다.

---

## A-2. 시연 1 실행 흐름

1. 입력 엑셀 파일을 읽는다.
2. 빈 행과 불필요한 행을 제거한다.
3. 대분류/중분류 공란을 위 값으로 채운다.
4. 시나리오 코드표를 읽는다.
5. 입력 데이터와 코드표를 `시나리오 식별자` 기준으로 연결한다.
6. 시나리오 식별자 기준으로 정렬한다.
7. 보기용으로 반복되는 시나리오 식별자와 중분류를 빈칸 처리한다.
8. 출력용 빈 템플릿에 결과를 저장한다.
9. 결과 파일을 다시 읽어 정상 저장 여부를 확인한다.

## A-3. 시연 1 전체 소스코드

아래 코드는 `시연1_시험항목기준_to_시나리오기준_자동변환_실행완료.ipynb`에 들어가는 핵심 전체 코드입니다.

```python
from pathlib import Path
import shutil

import pandas as pd
from openpyxl import load_workbook
from openpyxl.styles import Alignment, Border, Font, PatternFill, Side
from openpyxl.utils import get_column_letter


# ============================================================
# 1. 파일 경로와 시트 이름 설정
# ============================================================

# 현재 노트북이 실행되는 폴더를 기준으로 파일을 찾습니다.
BASE_DIR = Path(".")

# 입력용 엑셀 파일
INPUT_FILE = BASE_DIR / "시연1_입력용_시험항목기준_템플릿.xlsx"

# 출력용 빈 템플릿 파일
OUTPUT_TEMPLATE_FILE = BASE_DIR / "시연1_출력용_빈_시나리오기준_템플릿.xlsx"

# 최종 결과 파일명
OUTPUT_FILE = BASE_DIR / "시연1_자동변환_결과_시나리오기준.xlsx"

# 사용할 시트 이름
INPUT_SHEET = "시험항목기준_입력"
OUTPUT_SHEET = "시나리오기준_출력"
CODE_SHEET = "코드표"

print("입력 파일:", INPUT_FILE)
print("출력 템플릿 파일:", OUTPUT_TEMPLATE_FILE)
print("결과 파일:", OUTPUT_FILE)


# ============================================================
# 2. 파일 존재 여부 확인
# ============================================================

if not INPUT_FILE.exists():
    raise FileNotFoundError(f"입력 파일을 찾을 수 없습니다: {INPUT_FILE}")

if not OUTPUT_TEMPLATE_FILE.exists():
    raise FileNotFoundError(f"출력 템플릿 파일을 찾을 수 없습니다: {OUTPUT_TEMPLATE_FILE}")

print("파일 확인 완료")


# ============================================================
# 3. 시험항목 기준 입력 시트 읽기
# ============================================================

raw_df = pd.read_excel(
    INPUT_FILE,
    sheet_name=INPUT_SHEET,
    header=3,
    usecols="A:K"
)

# 전체가 빈 행은 제거합니다.
raw_df = raw_df.dropna(how="all")

# 순번이 숫자인 행만 실제 시험항목 데이터로 사용합니다.
raw_df = raw_df[pd.to_numeric(raw_df["순번"], errors="coerce").notna()].copy()

# 순번을 정수형으로 변환합니다.
raw_df["순번"] = raw_df["순번"].astype(int)

print("읽어온 행 수:", len(raw_df))
display(raw_df.head(10))


# ============================================================
# 4. 대분류/중분류 공란 채우기
# ============================================================

normalized_df = raw_df.copy()

# 엑셀에서 보기 좋게 비워둔 대분류/중분류를 위 값으로 채웁니다.
normalized_df["대분류"] = normalized_df["대분류"].ffill()
normalized_df["중분류"] = normalized_df["중분류"].ffill()

# 문자열 컬럼의 앞뒤 공백을 정리합니다.
text_columns = normalized_df.select_dtypes(include="object").columns
for col in text_columns:
    normalized_df[col] = normalized_df[col].astype("string").str.strip()

print("대분류/중분류 빈칸 채우기 완료")
display(normalized_df.head(10))


# ============================================================
# 5. 시나리오 코드표 읽기
# ============================================================

scenario_code_df = pd.read_excel(
    OUTPUT_TEMPLATE_FILE,
    sheet_name=CODE_SHEET,
    header=0,
    usecols="A:C"
)

# 전체가 빈 행 제거
scenario_code_df = scenario_code_df.dropna(how="all")

# 시나리오 식별자가 없는 행은 제외
scenario_code_df = scenario_code_df.dropna(subset=["시나리오 식별자"])

# 문자열 앞뒤 공백 정리
for col in scenario_code_df.columns:
    scenario_code_df[col] = scenario_code_df[col].astype("string").str.strip()

print("코드표 행 수:", len(scenario_code_df))
display(scenario_code_df)


# ============================================================
# 6. 입력 데이터와 시나리오 코드표 연결
# ============================================================

merged_df = normalized_df.merge(
    scenario_code_df,
    on="시나리오 식별자",
    how="left"
)

# 코드표에 없는 시나리오 식별자가 있는지 확인합니다.
missing_scenario_ids = (
    merged_df.loc[merged_df["시나리오명"].isna(), "시나리오 식별자"]
    .dropna()
    .unique()
)

if len(missing_scenario_ids) > 0:
    print("코드표에 없는 시나리오 식별자가 있습니다.")
    print(missing_scenario_ids)
else:
    print("모든 시나리오 식별자가 코드표와 정상 연결되었습니다.")

display(merged_df.head(10))


# ============================================================
# 7. 시나리오 기준 출력 데이터 만들기
# ============================================================

# 시나리오 식별자 → 시험항목 순번 순서로 정렬합니다.
output_df = merged_df.sort_values(
    by=["시나리오 식별자", "순번"],
    ascending=[True, True]
).reset_index(drop=True)

# 출력순번은 결과 파일에서 보기 편하도록 새로 부여합니다.
output_df.insert(0, "출력순번", range(1, len(output_df) + 1))

# 출력 템플릿 컬럼명에 맞게 컬럼을 구성합니다.
output_df["시험항목 순번"] = output_df["순번"]

output_df = output_df[
    [
        "출력순번",
        "시나리오 식별자",
        "시나리오명",
        "상황/목적",
        "시험항목 순번",
        "중분류",
        "소분류",
        "평가기준",
        "우선순위",
        "담당부서",
        "비고",
    ]
]

print("시나리오 기준 출력 데이터 행 수:", len(output_df))
display(output_df.head(20))


# ============================================================
# 8. 엑셀 표시용 데이터 만들기
# ============================================================

# 계산용 데이터는 그대로 두고, 엑셀에 보여줄 데이터만 따로 만듭니다.
display_df = output_df.copy()

# 바로 위 행과 시나리오 식별자가 같으면 True입니다.
same_scenario_as_previous_row = display_df["시나리오 식별자"].eq(
    display_df["시나리오 식별자"].shift()
)

# 같은 시나리오의 두 번째 행부터는 시나리오 관련 컬럼을 빈칸으로 표시합니다.
scenario_display_columns = ["시나리오 식별자", "시나리오명", "상황/목적"]
display_df.loc[same_scenario_as_previous_row, scenario_display_columns] = ""

# 같은 시나리오 안에서 중분류도 반복되면 두 번째 행부터 빈칸으로 표시합니다.
same_middle_category_as_previous_row = (
    same_scenario_as_previous_row
    & display_df["중분류"].eq(display_df["중분류"].shift())
)
display_df.loc[same_middle_category_as_previous_row, "중분류"] = ""

print("엑셀 표시용 데이터 미리보기")
display(display_df.head(25))


# ============================================================
# 9. 출력 템플릿에 결과 저장
# ============================================================

# 출력 템플릿 파일을 복사해서 결과 파일을 만듭니다.
shutil.copy(OUTPUT_TEMPLATE_FILE, OUTPUT_FILE)

# 복사된 결과 파일을 엽니다.
wb = load_workbook(OUTPUT_FILE)
ws = wb[OUTPUT_SHEET]

# 결과가 들어갈 시작 위치를 지정합니다.
START_ROW = 5
START_COL = 1

# 출력 컬럼 개수
num_cols = len(display_df.columns)

# 기존 출력 영역을 비웁니다.
for row in ws.iter_rows(
    min_row=START_ROW,
    max_row=ws.max_row,
    min_col=START_COL,
    max_col=num_cols
):
    for cell in row:
        cell.value = None

# 표시용 데이터를 엑셀 셀에 입력합니다.
for r_idx, row_values in enumerate(display_df.itertuples(index=False, name=None), start=START_ROW):
    for c_idx, value in enumerate(row_values, start=START_COL):
        if pd.isna(value):
            ws.cell(row=r_idx, column=c_idx).value = None
        else:
            ws.cell(row=r_idx, column=c_idx).value = value

# 보기 좋은 기본 서식을 적용합니다.
last_row = START_ROW + len(display_df) - 1
last_col = num_cols

thin_border = Border(
```

```python
BASE_DIR = Path(".")
```

현재 노트북이 실행되는 폴더를 의미합니다. 엑셀 파일과 노트북을 같은 폴더에 넣으면 파일 경로를 복잡하게 쓰지 않아도 됩니다.

---

### `pd.read_excel()`

```python
raw_df = pd.read_excel(
    INPUT_FILE,
    sheet_name=INPUT_SHEET,
    header=3,
    usecols="A:K"
)
```

엑셀 파일을 pandas DataFrame으로 읽습니다.

| 옵션 | 의미 |
|---|---|
| `INPUT_FILE` | 읽을 엑셀 파일 |
| `sheet_name=INPUT_SHEET` | 읽을 시트 이름 |
| `header=3` | 엑셀의 4번째 행을 컬럼명으로 사용 |
| `usecols="A:K"` | A열부터 K열까지만 읽기 |

파이썬은 숫자를 0부터 세기 때문에 `header=3`은 엑셀의 4번째 행입니다.

---

### `dropna(how="all")`

```python
raw_df = raw_df.dropna(how="all")
```

모든 칸이 비어 있는 행을 제거합니다.

---

### `pd.to_numeric()`

```python
raw_df = raw_df[pd.to_numeric(raw_df["순번"], errors="coerce").notna()].copy()
```

`순번`이 숫자인 행만 남깁니다. 엑셀에는 제목, 안내문, 빈 줄이 섞여 있을 수 있으므로 실제 데이터 행만 골라내는 과정입니다.

---

### `ffill()`

```python
normalized_df["대분류"] = normalized_df["대분류"].ffill()
normalized_df["중분류"] = normalized_df["중분류"].ffill()
```

현업 엑셀에서 보기 좋게 비워둔 대분류와 중분류를 위 값으로 채웁니다.

이 과정이 없으면 2행, 3행의 대분류/중분류가 빈값으로 남아서 정렬이나 그룹핑할 때 문제가 생깁니다.

---

### `merge()`

```python
merged_df = normalized_df.merge(
    scenario_code_df,
    on="시나리오 식별자",
    how="left"
)
```

입력 데이터와 코드표를 연결합니다.

입력 데이터에는 시나리오 식별자만 있고, 코드표에는 시나리오명과 상황/목적이 있습니다. `merge()`를 사용하면 시나리오 식별자를 기준으로 시나리오명을 붙일 수 있습니다.

---

### `sort_values()`

```python
output_df = merged_df.sort_values(
    by=["시나리오 식별자", "순번"],
    ascending=[True, True]
)
```

시나리오 기준 출력 파일이므로 먼저 `시나리오 식별자`로 정렬하고, 같은 시나리오 안에서는 `순번`으로 정렬합니다.

---

### 보기용 공란 처리

```python
same_scenario_as_previous_row = display_df["시나리오 식별자"].eq(
    display_df["시나리오 식별자"].shift()
)

display_df.loc[same_scenario_as_previous_row, ["시나리오 식별자", "시나리오명", "상황/목적"]] = ""
```

바로 위 행과 시나리오 식별자가 같으면 같은 묶음입니다. 따라서 엑셀에서는 두 번째 행부터 시나리오 식별자, 시나리오명, 상황/목적을 빈칸으로 보여줍니다.

중요한 점은 이것이 **계산용 데이터가 아니라 보기용 데이터**라는 점입니다.

## A-5. 시연 1 함수 버전

실무에서는 매번 긴 코드를 실행하기보다 함수로 만들어 두는 것이 좋습니다.

```python
def convert_test_item_to_scenario_file(
    input_file,
    output_template_file,
    output_file,
    input_sheet="시험항목기준_입력",
    output_sheet="시나리오기준_출력",
    code_sheet="코드표",
):
    """
    시험항목 기준 입력 엑셀을 시나리오 기준 출력 엑셀로 변환합니다.
    """

    input_file = Path(input_file)
    output_template_file = Path(output_template_file)
    output_file = Path(output_file)

    if not input_file.exists():
        raise FileNotFoundError(f"입력 파일을 찾을 수 없습니다: {input_file}")

    if not output_template_file.exists():
        raise FileNotFoundError(f"출력 템플릿 파일을 찾을 수 없습니다: {output_template_file}")

    raw_df = pd.read_excel(
        input_file,
        sheet_name=input_sheet,
        header=3,
        usecols="A:K"
    )
    raw_df = raw_df.dropna(how="all")
    raw_df = raw_df[pd.to_numeric(raw_df["순번"], errors="coerce").notna()].copy()
    raw_df["순번"] = raw_df["순번"].astype(int)

    normalized_df = raw_df.copy()
    normalized_df["대분류"] = normalized_df["대분류"].ffill()
    normalized_df["중분류"] = normalized_df["중분류"].ffill()

    text_columns = normalized_df.select_dtypes(include="object").columns
    for col in text_columns:
        normalized_df[col] = normalized_df[col].astype("string").str.strip()

    scenario_code_df = pd.read_excel(
        output_template_file,
        sheet_name=code_sheet,
        header=0,
        usecols="A:C"
    )
    scenario_code_df = scenario_code_df.dropna(how="all")
    scenario_code_df = scenario_code_df.dropna(subset=["시나리오 식별자"])

    for col in scenario_code_df.columns:
        scenario_code_df[col] = scenario_code_df[col].astype("string").str.strip()

    merged_df = normalized_df.merge(
        scenario_code_df,
        on="시나리오 식별자",
        how="left"
    )

    output_df = merged_df.sort_values(
        by=["시나리오 식별자", "순번"],
        ascending=[True, True]
    ).reset_index(drop=True)

    output_df.insert(0, "출력순번", range(1, len(output_df) + 1))
    output_df["시험항목 순번"] = output_df["순번"]

    output_df = output_df[
        [
            "출력순번",
            "시나리오 식별자",
            "시나리오명",
            "상황/목적",
            "시험항목 순번",
            "중분류",
            "소분류",
            "평가기준",
            "우선순위",
            "담당부서",
            "비고",
        ]
    ]

    display_df = output_df.copy()

    same_scenario_as_previous_row = display_df["시나리오 식별자"].eq(
        display_df["시나리오 식별자"].shift()
    )

    scenario_display_columns = ["시나리오 식별자", "시나리오명", "상황/목적"]
    display_df.loc[same_scenario_as_previous_row, scenario_display_columns] = ""

    same_middle_category_as_previous_row = (
        same_scenario_as_previous_row
        & display_df["중분류"].eq(display_df["중분류"].shift())
    )
    display_df.loc[same_middle_category_as_previous_row, "중분류"] = ""

    shutil.copy(output_template_file, output_file)

    wb = load_workbook(output_file)
    ws = wb[output_sheet]

    start_row = 5
    start_col = 1
    num_cols = len(display_df.columns)

    for row in ws.iter_rows(
        min_row=start_row,
        max_row=ws.max_row,
        min_col=start_col,
        max_col=num_cols
    ):
        for cell in row:
            cell.value = None

    for r_idx, row_values in enumerate(display_df.itertuples(index=False, name=None), start=start_row):
        for c_idx, value in enumerate(row_values, start=start_col):
            if pd.isna(value):
                ws.cell(row=r_idx, column=c_idx).value = None
            else:
                ws.cell(row=r_idx, column=c_idx).value = value

    wb.save(output_file)

    return output_df
```

```python
    input_file="시연1_입력용_시험항목기준_템플릿.xlsx",
    output_template_file="시연1_출력용_빈_시나리오기준_템플릿.xlsx",
```

# PART B. 시연 2 — 시나리오 기준 → 시험항목 기준

## B-1. 시연 2의 업무 상황

입력 파일은 시나리오 기준으로 작성되어 있습니다.

예를 들어 아래와 같은 형태입니다.

| 시나리오 식별자 | 시나리오명 | 순번 | 시험항목ID | 소분류 |
|---|---|---:|---|---|
| SCN-MOB-001 | 기동 운용 시나리오 | 1 | TC-001 | 저속 주행 |
|  |  | 2 | TC-002 | 중속 주행 |
| SCN-COM-001 | 통신망 구성 시나리오 | 3 | TC-003 | 통신 연결 |

이 데이터를 시험항목 기준으로 바꾸면 아래처럼 됩니다.

| 순번 | 대분류 | 중분류 | 시험항목ID | 소분류 | 시나리오 식별자 |
|---:|---|---|---|---|---|
| 1 | 운용환경 | 기동조건 | TC-001 | 저속 주행 | SCN-MOB-001 |
| 2 |  |  | TC-002 | 중속 주행 | SCN-MOB-001 |
| 3 |  | 통신조건 | TC-003 | 통신 연결 | SCN-COM-001 |

이때 시나리오 기준 입력 파일에는 대분류, 평가기준, 판정방식 같은 시험항목 상세 정보가 부족할 수 있습니다. 그래서 `시험항목마스터` 시트를 함께 사용합니다.

---

## B-2. 시연 2 실행 흐름

1. 시나리오 기준 입력 시트를 읽는다.
2. 시험항목 마스터 시트를 읽는다.
3. 빈 행과 불필요한 행을 제거한다.
4. 시나리오 식별자, 시나리오명, 운용조건 공란을 위 값으로 채운다.
5. 시나리오 입력 데이터와 시험항목 마스터를 `순번`, `시험항목ID` 기준으로 연결한다.
6. 시험항목 기준 컬럼을 구성한다.
7. 순번 기준으로 정렬한다.
8. 보기용으로 반복되는 대분류, 중분류, 시험항목 정보를 빈칸 처리한다.
9. 출력용 빈 템플릿에 결과를 저장한다.
10. 검증용 시트를 추가한다.

---

## B-3. 시연 2 전체 소스코드

아래 코드는 `시연2_시나리오기준_to_시험항목기준_자동변환.ipynb`에 들어가는 핵심 전체 코드입니다.

```python

import pandas as pd
from openpyxl import load_workbook
from pathlib import Path

print("라이브러리 로드 완료")


# ============================================================
# 1. 파일 경로 설정
# ============================================================

input_file = Path("시연2_입력용_시나리오기준_템플릿.xlsx")
output_template_file = Path("시연2_출력용_빈_시험항목기준_템플릿.xlsx")
result_file = Path("시연2_자동변환_결과_시험항목기준_파이썬실행.xlsx")

print("입력 파일:", input_file)
print("출력 템플릿:", output_template_file)
print("결과 파일:", result_file)


# ============================================================
# 2. 입력 데이터 읽기
# ============================================================

scenario_df = pd.read_excel(
    input_file,
    sheet_name="시나리오기준_입력",
    header=3
)

master_df = pd.read_excel(
    input_file,
    sheet_name="시험항목마스터",
    header=3
)

print("시나리오 기준 입력 데이터 크기:", scenario_df.shape)
print("시험항목 마스터 데이터 크기:", master_df.shape)

display(scenario_df.head(10))


# ============================================================
# 3. 불필요한 빈 행 제거
# ============================================================

scenario_df = scenario_df.dropna(how="all")
master_df = master_df.dropna(how="all")

# 순번이 숫자인 행만 남기기
scenario_df = scenario_df[pd.to_numeric(scenario_df["순번"], errors="coerce").notna()].copy()
master_df = master_df[pd.to_numeric(master_df["순번"], errors="coerce").notna()].copy()

# 순번을 정수형으로 변환
scenario_df["순번"] = scenario_df["순번"].astype(int)
master_df["순번"] = master_df["순번"].astype(int)

print("정리 후 시나리오 입력 행 수:", len(scenario_df))
print("정리 후 마스터 행 수:", len(master_df))

display(scenario_df.head(10))


# ============================================================
# 4. 시나리오 기준 공란 채우기
# ============================================================

# 이번 방향에서는 대분류/중분류가 아니라 시나리오 정보를 채웁니다.
fill_cols = [
    "시나리오 식별자",
    "시나리오명",
    "상황/운용조건",
    "시나리오 목적",
    "중분류"

scenario_df[fill_cols] = scenario_df[fill_cols].ffill()

print("공란 채우기 완료")
display(scenario_df.head(10))


# ============================================================
# 5. 시나리오 입력 데이터와 시험항목 마스터 연결
# ============================================================

merged_df = scenario_df.merge(
    master_df,
    on=["순번", "시험항목ID"],
    how="left",
    suffixes=("_입력", "")
)

print("merge 후 데이터 크기:", merged_df.shape)

# 마스터에서 찾지 못한 시험항목이 있는지 확인합니다.
missing_master = merged_df[merged_df["대분류"].isna()]

if len(missing_master) == 0:
    print("모든 시험항목이 시험항목마스터와 정상 연결되었습니다.")
else:
    print("주의: 마스터에서 찾지 못한 시험항목이 있습니다.")
    display(missing_master[["순번", "시험항목ID", "소분류_입력", "시나리오 식별자"]])

display(merged_df.head(10))


# ============================================================
# 6. 최종 출력 컬럼 구성
# ============================================================

output_columns = [
    "순번",
    "대분류",
    "중분류",
    "시험항목ID",
    "소분류",
    "시험항목명",
    "평가기준",
    "시나리오 식별자",
    "시나리오명",
    "판정방식",
    "중요도",
    "담당조직",
    "비고"
]

# 입력표 쪽 비고가 있으면 사용하고, 없으면 빈 문자열로 처리합니다.
if "비고_입력" in merged_df.columns:
    merged_df["비고"] = merged_df["비고_입력"].fillna("")
else:
    merged_df["비고"] = ""

result_df = merged_df[output_columns].copy()

# 시험항목 기준이므로 순번을 먼저 기준으로 정렬합니다.
result_df = result_df.sort_values(["순번", "시나리오 식별자"]).reset_index(drop=True)

print("최종 결과 행 수:", len(result_df))
display(result_df.head(15))


# ============================================================
# 7. 보기용 출력 데이터 만들기
# ============================================================

display_df = result_df.copy()

# 같은 대분류/중분류가 바로 위 행과 같으면 보기용으로 빈칸 처리합니다.
display_df.loc[display_df["대분류"].eq(display_df["대분류"].shift()), "대분류"] = ""
display_df.loc[display_df["중분류"].eq(display_df["중분류"].shift()), "중분류"] = ""

# 같은 시험항목ID가 바로 위 행과 같다는 것은
# 하나의 시험항목이 여러 시나리오에 연결되었다는 뜻입니다.
same_item = display_df["시험항목ID"].eq(display_df["시험항목ID"].shift())

blank_cols_when_same_item = [
    "순번",
    "시험항목ID",
    "소분류",
    "시험항목명",
    "평가기준",
    "판정방식",
    "중요도",
    "담당조직"
]

display_df.loc[same_item, blank_cols_when_same_item] = ""

print("보기용 출력 데이터")
display(display_df.head(20))


# ============================================================
# 8. 출력 템플릿에 결과 쓰기
# ============================================================

wb = load_workbook(output_template_file)
ws = wb["시험항목기준_출력"]

start_row = 5
start_col = 1

# 기존 데이터 영역 지우기
for row in ws.iter_rows(
    min_row=start_row,
    max_row=ws.max_row,
    min_col=start_col,
    max_col=len(output_columns)
):
    for cell in row:
        cell.value = None

# 보기용 데이터 쓰기
for r_idx, row in enumerate(display_df[output_columns].itertuples(index=False), start=start_row):
    for c_idx, value in enumerate(row, start=start_col):
        ws.cell(row=r_idx, column=c_idx).value = value

# 결과 파일 저장
wb.save(result_file)

print("출력 템플릿에 결과 저장 완료:", result_file)


# ============================================================
# 9. 데이터 검증용 시트 추가
# ============================================================

with pd.ExcelWriter(
    result_file,
    engine="openpyxl",
    mode="a",
    if_sheet_exists="replace"
) as writer:
    result_df.to_excel(writer, sheet_name="데이터용_시험항목기준", index=False)
    scenario_df.to_excel(writer, sheet_name="정규화_시나리오기준", index=False)

print("검증용 시트 추가 완료")
```

## B-4. 시연 2 코드 해설

### 시나리오 입력과 시험항목 마스터를 함께 읽는 이유

```python
scenario_df = pd.read_excel(input_file, sheet_name="시나리오기준_입력", header=3)
master_df = pd.read_excel(input_file, sheet_name="시험항목마스터", header=3)
```

시나리오 기준 입력표에는 시나리오 중심 정보가 들어 있습니다. 하지만 시험항목 기준 출력표에는 대분류, 소분류, 시험항목명, 평가기준 같은 시험항목 상세 정보가 필요합니다.

따라서 시험항목 상세 정보는 `시험항목마스터` 시트에서 가져옵니다.

---

### 이번에는 시나리오 정보를 `ffill()` 한다

```python
fill_cols = [
    "시나리오 식별자",
    "시나리오명",
    "상황/운용조건",
    "시나리오 목적",
    "중분류"
]

scenario_df[fill_cols] = scenario_df[fill_cols].ffill()
```

시연 1에서는 대분류와 중분류를 채웠습니다. 시연 2에서는 입력 파일이 시나리오 기준이기 때문에 시나리오 식별자, 시나리오명, 상황/운용조건 같은 시나리오 정보를 채웁니다.

---

### `merge()` 기준이 다르다

```python
merged_df = scenario_df.merge(
    master_df,
    on=["순번", "시험항목ID"],
    how="left",
    suffixes=("_입력", "")
)
```

시연 1에서는 `시나리오 식별자`로 코드표를 붙였습니다.

시연 2에서는 시나리오 입력 데이터와 시험항목 마스터를 연결해야 하므로 `순번`, `시험항목ID` 두 컬럼을 기준으로 붙입니다.

실무에서는 `시험항목ID`가 고유하다면 `시험항목ID` 하나만 기준으로 연결하는 것도 가능합니다.

---

### 정렬 기준이 반대가 된다

```python
result_df = result_df.sort_values(["순번", "시나리오 식별자"]).reset_index(drop=True)
```

시연 2의 출력은 시험항목 기준입니다. 따라서 먼저 `순번`으로 정렬합니다.

같은 시험항목이 여러 시나리오에 포함될 수 있으므로, 같은 순번 안에서는 `시나리오 식별자` 순서로 정렬합니다.

---

### 같은 시험항목이 반복될 수 있다

```python
same_item = display_df["시험항목ID"].eq(display_df["시험항목ID"].shift())
```

하나의 시험항목이 여러 시나리오에 연결되면 시험항목 정보가 반복됩니다. 엑셀에서는 보기 좋게 두 번째 행부터 시험항목 정보를 빈칸으로 처리합니다.

## B-5. 시연 2 함수 버전

```python
def convert_scenario_to_test_item(input_file, output_template_file, result_file):
    """시나리오 기준 입력 엑셀을 시험항목 기준 결과 엑셀로 변환합니다."""

    input_file = Path(input_file)
    output_template_file = Path(output_template_file)
    result_file = Path(result_file)

    # 1. 입력 읽기
    scenario_df = pd.read_excel(input_file, sheet_name="시나리오기준_입력", header=3)
    master_df = pd.read_excel(input_file, sheet_name="시험항목마스터", header=3)

    # 2. 빈 행 제거 및 순번 정리
    scenario_df = scenario_df.dropna(how="all")
    master_df = master_df.dropna(how="all")

    scenario_df = scenario_df[pd.to_numeric(scenario_df["순번"], errors="coerce").notna()].copy()
    master_df = master_df[pd.to_numeric(master_df["순번"], errors="coerce").notna()].copy()

    scenario_df["순번"] = scenario_df["순번"].astype(int)
    master_df["순번"] = master_df["순번"].astype(int)

    # 3. 시나리오 기준 공란 채우기
    fill_cols = ["시나리오 식별자", "시나리오명", "상황/운용조건", "시나리오 목적", "중분류"]
    scenario_df[fill_cols] = scenario_df[fill_cols].ffill()

    # 4. 시험항목마스터 연결
    merged_df = scenario_df.merge(
        master_df,
        on=["순번", "시험항목ID"],
        how="left",
        suffixes=("_입력", "")
    )

    if "비고_입력" in merged_df.columns:
        merged_df["비고"] = merged_df["비고_입력"].fillna("")
    else:
        merged_df["비고"] = ""

    output_columns = [
        "순번", "대분류", "중분류", "시험항목ID", "소분류", "시험항목명",
        "평가기준", "시나리오 식별자", "시나리오명", "판정방식", "중요도", "담당조직", "비고"
    ]

    result_df = merged_df[output_columns].copy()
    result_df = result_df.sort_values(["순번", "시나리오 식별자"]).reset_index(drop=True)

    # 5. 보기용 출력 데이터 생성
    display_df = result_df.copy()
    display_df.loc[display_df["대분류"].eq(display_df["대분류"].shift()), "대분류"] = ""
    display_df.loc[display_df["중분류"].eq(display_df["중분류"].shift()), "중분류"] = ""

    same_item = display_df["시험항목ID"].eq(display_df["시험항목ID"].shift())
    blank_cols_when_same_item = [
        "순번", "시험항목ID", "소분류", "시험항목명",
        "평가기준", "판정방식", "중요도", "담당조직"
    ]
    display_df.loc[same_item, blank_cols_when_same_item] = ""

    # 6. 템플릿에 쓰기
    wb = load_workbook(output_template_file)
    ws = wb["시험항목기준_출력"]

    start_row = 5
    start_col = 1

    for row in ws.iter_rows(
        min_row=start_row,
        max_row=ws.max_row,
        min_col=start_col,
        max_col=len(output_columns)
    ):
        for cell in row:
            cell.value = None

    for r_idx, row in enumerate(display_df[output_columns].itertuples(index=False), start=start_row):
        for c_idx, value in enumerate(row, start=start_col):
            ws.cell(row=r_idx, column=c_idx).value = value

    wb.save(result_file)

    # 7. 검증용 시트 추가
    with pd.ExcelWriter(
        result_file,
        engine="openpyxl",
        mode="a",
        if_sheet_exists="replace"
    ) as writer:
        result_df.to_excel(writer, sheet_name="데이터용_시험항목기준", index=False)
        scenario_df.to_excel(writer, sheet_name="정규화_시나리오기준", index=False)

    return result_df
```

함수 실행 예시는 다음과 같습니다.

```python
result_df = convert_scenario_to_test_item(
    input_file="시연2_입력용_시나리오기준_템플릿.xlsx",
    output_template_file="시연2_출력용_빈_시험항목기준_템플릿.xlsx",
    result_file="시연2_자동변환_결과_시험항목기준_파이썬실행.xlsx"
)

print("최종 결과 행 수:", len(result_df))
display(result_df.head(10))
```

# PART C. 두 시연을 비교해서 설명하는 방법

## C-1. 수업에서 강조할 핵심 메시지

이번 실습의 핵심은 엑셀을 예쁘게 만드는 것이 아니라, **데이터 기준을 바꾸는 것**입니다.

| 관점 | 설명 |
|---|---|
| 시험항목 기준 | 각 시험항목이 어떤 시나리오에 포함되는지 관리 |
| 시나리오 기준 | 각 시나리오가 어떤 시험항목들로 구성되는지 관리 |

두 표는 서로 다른 데이터가 아니라, 같은 데이터를 다른 기준으로 바라본 것입니다.

---

## C-2. 둘 다 가능한 이유

시험항목과 시나리오 사이에는 연결 관계가 있습니다.

```text
시험항목 1개  → 여러 시나리오에 포함될 수 있음
시나리오 1개 → 여러 시험항목을 포함할 수 있음
```

즉, 실제 구조는 아래와 같은 매칭표입니다.

| 시험항목ID | 시나리오 식별자 |
|---|---|
| TC-001 | SCN-MOB-001 |
| TC-002 | SCN-MOB-001 |
| TC-003 | SCN-COM-001 |
| TC-003 | SCN-OPS-002 |

파이썬은 이 매칭표를 기준으로 정렬 방향만 바꿔서 두 가지 형태의 엑셀을 만들 수 있습니다.

---

## C-3. 실무적으로 더 안전한 구조

현업에서는 처음부터 보기용 엑셀만 관리하면 자동화가 어려워집니다.

더 안전한 방식은 아래와 같습니다.

1. 시험항목 마스터를 만든다.
2. 시나리오 마스터를 만든다.
3. 시험항목-시나리오 매칭표를 만든다.
4. 필요한 출력 양식은 파이썬으로 자동 생성한다.

추천 구조는 다음과 같습니다.

| 시트명 | 역할 |
|---|---|
| 시험항목마스터 | 시험항목ID, 대분류, 중분류, 소분류, 평가기준 관리 |
| 시나리오마스터 | 시나리오 식별자, 시나리오명, 운용조건, 목적 관리 |
| 매칭표 | 시험항목ID와 시나리오 식별자의 연결 관계 관리 |
| 시험항목기준_출력 | 자동 생성 결과 |
| 시나리오기준_출력 | 자동 생성 결과 |

# PART E. 강의 시연 멘트 예시

## E-1. 도입 멘트

> 오늘은 실무에서 많이 발생하는 엑셀 자동화 문제를 다룹니다.  
> 같은 시험 데이터를 시험항목 기준으로 볼 수도 있고, 시나리오 기준으로 볼 수도 있습니다.  
> 사람이 수작업으로 바꾸면 시간이 오래 걸리고 실수가 생기지만, pandas를 사용하면 기준만 바꿔서 자동 변환할 수 있습니다.

---

## E-2. `ffill()` 설명 멘트

> 엑셀에서는 보기 좋게 만들기 위해 같은 대분류나 중분류를 반복해서 쓰지 않는 경우가 많습니다.  
> 하지만 파이썬은 빈칸을 보고 위 값과 같다고 자동으로 이해하지 못합니다.  
> 그래서 먼저 `ffill()`을 사용해서 빈칸을 위 값으로 채워 줍니다.  
> 이 작업을 해야 데이터가 정상적으로 정렬되고 그룹핑됩니다.

---

## E-3. `merge()` 설명 멘트

> `merge()`는 엑셀의 VLOOKUP과 비슷하다고 생각하면 됩니다.  
> 기준 컬럼이 같은 행끼리 서로 붙여 주는 기능입니다.  
> 여기서는 시나리오 식별자를 기준으로 시나리오명을 붙이거나, 시험항목ID를 기준으로 시험항목 상세 정보를 붙입니다.

---

## E-4. 정렬 설명 멘트

> 시연 1에서는 시나리오 기준으로 보고 싶기 때문에 시나리오 식별자를 먼저 기준으로 정렬합니다.

# PART G. 최종 정리

이번 실습에서 배운 핵심은 다음과 같습니다.

| 핵심 기능 | 설명 |
|---|---|
