1. 왜 데이터베이스가 필요한가?

CSV와 엑셀은 실습에는 편리하지만, 실무에서는 데이터가 계속 쌓이고 여러 사람이 동시에 사용합니다. 데이터베이스는 이런 데이터를 안정적으로 저장하고 검색하기 위한 시스템입니다.

저장데이터를 구조적으로 보관합니다.
검색SQL로 필요한 데이터만 가져옵니다.
관리여러 테이블과 관계를 관리합니다.
분석 연결Python과 pandas로 불러와 분석합니다.

2. 테이블과 SQL

데이터베이스의 테이블은 DataFrame과 비슷하게 행과 열로 구성됩니다. SQL은 데이터베이스에 질문하는 언어입니다.

3. sqlite3로 데이터베이스 맛보기

sqlite3는 별도 서버 없이 파일 하나로 사용할 수 있는 가벼운 데이터베이스입니다.

import sqlite3 import pandas as pd conn = sqlite3.connect("sample.db") sales = pd.DataFrame({ "menu": ["아메리카노", "카페라떼", "샌드위치"], "sales": [54000, 40000, 32500] }) sales.to_sql("sales", conn, if_exists="replace", index=False)

4. SQL로 데이터 가져오기

pd.read_sql()을 사용하면 SQL 실행 결과를 DataFrame으로 가져올 수 있습니다.

query = "SELECT * FROM sales WHERE sales >= 40000" result = pd.read_sql(query, conn) print(result) conn.close()
핵심: SQL로 필요한 데이터를 먼저 줄여 가져오면 Python에서 처리해야 할 데이터 양을 줄일 수 있습니다.

MySQL은 실제 서비스와 업무 시스템에서 많이 사용하는 서버형 데이터베이스입니다. 이 과정에서는 맛보기만 다루고, 이후 SQL 과정에서 테이블 설계, 조인, 집계, 인덱스 등을 깊게 다룰 수 있습니다.

용도실습, 작은 프로젝트실무 시스템
